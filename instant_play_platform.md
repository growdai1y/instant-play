# Instant Play Platform

영상 전송, 적응형 화질, DRM 및 일반적인 멀티미디어 재생 과정은 [텔레비전 플랫폼 기반 주문형 비디오 스트리밍 구조와 처리 흐름](./vod_app_platform_streaming.md) 문서에서 설명한다. 본 문서는 [Instant Play 플랫폼 도면](./instant_play_platform.drawio)에 표시된 구성과 연결관계를 기준으로, 광고 시청 완료를 VOD 콘텐츠의 한시적 재생권한 획득 조건으로 사용하는 즉시 재생 시스템을 설명한다.

도면에서 광고 처리부와 콘텐츠 재생권한 획득부는 즉시 재생 플랫폼에 배치되고, 사용자 인터페이스·즉시 재생 제어부·광고 시청 확인부·멀티미디어 재생기는 사용자 단말에 배치된다. 이하의 구성요소 명칭, 소속 및 상호작용은 이 배치를 기준으로 한다.

---

## 1. 정의와 핵심 개념

### 1.1 즉시 재생

**Instant Play(이하 “즉시 재생”)**는 사용자가 VOD 업체의 앱을 설치하거나 해당 업체 계정으로 로그인하지 않고도, 사용자 단말에서 선택한 콘텐츠에 요구되는 광고의 시청 완료가 확인된 후 VOD 업체가 발급한 한시적 콘텐츠 재생권한을 이용하여 콘텐츠를 재생하는 서비스이다.

~~~text
VOD 업체·콘텐츠 등록
→ 등록 콘텐츠의 실제 재생 가능성 검증
→ 검증된 콘텐츠를 추천 시스템에 등록
→ 사용자가 사용자 단말에서 즉시 재생 콘텐츠 선택
→ 콘텐츠에 대응하는 필수 광고정보 획득
→ 사용자 단말의 멀티미디어 재생기로 광고 재생
→ 사용자 단말의 광고 시청 확인부와 플랫폼의 광고 처리부가 광고 완료 확인
→ 확인된 경우에만 콘텐츠 재생권한 획득 요청
→ 플랫폼이 등록정보를 이용하여 VOD 업체 시스템에 재생권한 요청
→ VOD 업체가 발급한 한시적 콘텐츠 재생권한 획득
→ 사용자 단말의 멀티미디어 재생기로 VOD 콘텐츠 재생
~~~

### 1.2 핵심 명제

> 광고 시청 완료는 플랫폼이 임의로 VOD 콘텐츠의 재생권한을 생성하는 근거가 아니라, 플랫폼의 콘텐츠 재생권한 획득부가 등록된 VOD 업체 시스템에 한시적 콘텐츠 재생권한을 요청하도록 허용하는 선행조건이다.

권한 발급 주체는 VOD 업체 시스템이다. 즉시 재생 플랫폼은 광고 시청 완료 확인과 VOD 업체의 권한 발급 사이를 연결하고, 사용자 단말은 플랫폼이 획득한 권한을 이용하여 콘텐츠를 재생한다.

### 1.3 주요 용어

- **업체·콘텐츠 등록정보:** VOD 업체 식별자, 콘텐츠 식별자, 콘텐츠 재생정보, 권한 요청방법 및 즉시 재생 제공조건을 포함하는 정보이다.
- **광고정보 또는 광고 목록:** 선택된 콘텐츠를 즉시 재생하기 전에 또는 재생 중에 사용자 단말에서 재생되어야 하는 하나 이상의 광고를 나타내는 정보이다. 광고를 선정하는 구체적인 알고리즘은 본 발명의 중심이 아니다.
- **광고 시청 확인정보:** 특정 즉시 재생 세션, 콘텐츠 및 광고 요구시점에 대응하는 필수 광고가 완료되었음을 사용자 단말의 광고 시청 확인부와 플랫폼의 광고 처리부 사이에서 확인한 정보이다.
- **콘텐츠 재생권한:** VOD 업체 시스템이 특정 콘텐츠, 세션, 재생구간 또는 유효시간에 한정하여 발급한 재생 허용정보이다.
- **콘텐츠 재생권한 획득:** 플랫폼이 등록된 업체·콘텐츠 정보와 광고 시청 확인정보를 이용하여 VOD 업체 시스템에 권한을 요청하고, 업체가 발급한 권한을 수신하는 처리이다.
- **즉시 재생 제어부:** 사용자 단말에서 광고정보 획득, 광고 재생, 광고 시청 확인, 콘텐츠 재생권한 획득 및 VOD 콘텐츠 재생의 순서를 제어하는 구성이다.
- **VOD 재생 검증:** 업체·콘텐츠 등록단계에서 등록된 권한 요청방법과 재생정보로 개별 콘텐츠가 실제 재생되는지를 시험하는 처리이다.

### 1.4 로그인 없음의 의미

“로그인 없음”은 사용자가 VOD 업체의 사용자 계정으로 로그인하지 않는다는 의미이다.

| 항목 | 필요 여부 |
|---|---|
| VOD 업체 앱 설치·실행 | 불필요 |
| VOD 업체 사용자 계정 로그인 | 불필요 |
| 플랫폼 또는 단말의 즉시 재생 세션 | 필요 |
| 필수 광고 시청 완료 확인 | 필요 |
| VOD 업체가 발급한 한시적 콘텐츠 재생권한 | 필요 |
| 기존 스트리밍·DRM·콘텐츠 보호 처리 | 해당 콘텐츠의 기존 방식 사용 |

콘텐츠의 암호화, 복호화, DRM 및 보호 출력은 VOD 업체 시스템과 사용자 단말이 기존에 지원하는 방식을 그대로 사용할 수 있다. 본 발명은 기존 콘텐츠 보호방식 자체를 변경하는 것이 아니라, 광고 시청 완료 확인과 업체가 발급하는 한시적 재생권한을 연결한다.

## 2. 일반 VOD 재생과 즉시 재생의 차이

| 구분 | 일반 VOD 앱 재생 | 즉시 재생 |
|---|---|---|
| 사용자 진입 | VOD 업체 앱 실행 | 추천 시스템이 제공한 콘텐츠를 사용자 단말에서 선택 |
| 사용자 조건 | 업체 로그인, 구독 또는 구매 | 필수 광고 시청 완료 확인 |
| 콘텐츠 노출 | 개별 VOD 업체가 관리 | 검증된 복수 업체 콘텐츠를 추천 시스템에 등록 가능 |
| 권한 발급 주체 | VOD 업체 시스템 | VOD 업체 시스템 |
| 플랫폼의 역할 | 앱 실행 또는 콘텐츠 노출 연결 | 업체·콘텐츠 관리, 재생 검증, 광고 처리, 권한 요청·획득 |
| 단말의 역할 | 업체 앱 또는 재생기 실행 | 즉시 재생 제어부가 광고 확인과 VOD 재생 순서를 제어 |
| 콘텐츠 재생 | 업체 앱 또는 기존 재생기 | 단말의 멀티미디어 재생기 |
| 콘텐츠 보호 | 업체의 기존 방식 | 업체의 기존 방식과 동일 |
| 광고수익 | 업체 정책에 따라 처리 | 즉시 재생 세션의 광고·권한·재생 기록을 업체 배분 근거로 사용 가능 |

즉시 재생의 차별점은 광고, 추천 또는 스트리밍 기능 각각에 있지 않다. 다음 기능을 하나의 기술적 제어관계로 결합하는 데 있다.

1. VOD 업체가 즉시 재생에 필요한 업체·콘텐츠·권한 요청정보를 플랫폼에 등록한다.
2. 플랫폼이 등록 콘텐츠의 권한 획득과 실제 재생 가능성을 검증한다.
3. 검증된 콘텐츠만 추천 시스템에 등록한다.
4. 사용자 단말이 선택 콘텐츠에 대응하는 필수 광고를 재생하고 플랫폼과 광고 완료를 확인한다.
5. 광고 완료가 확인된 경우에만 플랫폼이 VOD 업체 시스템에서 한시적 콘텐츠 재생권한을 획득한다.
6. 사용자 단말이 해당 권한으로 콘텐츠를 재생한다.

## 3. 도면을 기준으로 한 전체 시스템 구조

### 3.1 시스템 경계

도면의 즉시 재생 시스템은 다음 네 개의 시스템 영역으로 구분된다.

원본 Draw.io 도면에는 도면부호가 기재되어 있지 않으므로, 괄호 안의 번호는 본 문서에서 구성요소를 일관되게 참조하기 위해 부여한 것이다. 최종 특허도면을 작성할 때 동일한 번호를 원본 도면에 반영할 수 있다.

| 시스템 영역 | 도면부호 | 포함 구성 |
|---|---:|---|
| 즉시 재생 플랫폼 | 100 | VOD 업체·콘텐츠 관리부, VOD 재생 검증부, 추천 콘텐츠 등록부, 광고 처리부, 콘텐츠 재생권한 획득부, 즉시 재생 콘텐츠 저장소 |
| VOD 업체 시스템 | 200 | 업체·콘텐츠 등록정보 제공, 콘텐츠 재생권한 발급, VOD 콘텐츠 제공 |
| 사용자 단말 | 300 | 사용자 인터페이스, 즉시 재생 제어부, 광고 시청 확인부, 멀티미디어 재생기 |
| 추천 시스템 | 400 | 플랫폼이 등록한 즉시 재생 콘텐츠의 추천·노출 및 사용자 단말 제공 |

광고 처리부(140)는 즉시 재생 플랫폼(100)에 속하고, 광고 시청 확인부(330)는 사용자 단말(300)에 속한다. 광고 시청 확인은 이 두 구성의 연동으로 수행된다. 별도의 외부 광고 시스템이 본 발명의 필수 구성으로 요구되는 것은 아니다.

### 3.2 기능 흐름에 대응하는 시스템 구조

다음 구조는 `instant_play_platform.drawio`의 시스템 경계와 연결관계를 문서용으로 옮긴 것이다.

~~~mermaid
flowchart LR
    RECOMMEND["추천 시스템(400)"]
    VOD["VOD 업체 시스템(200)"]

    subgraph DEVICE["사용자 단말(300)"]
        direction TB
        UI["사용자 인터페이스(320)"]
        CTRL["즉시 재생 제어부(310)"]
        ADCHECK["광고 시청 확인부(330)"]
        PLAYER["멀티미디어 재생기(340)"]

        UI -->|"즉시 재생 콘텐츠 정보"| CTRL
        CTRL <-->|"광고 확인 요청·결과"| ADCHECK
        CTRL -->|"광고·콘텐츠 재생"| PLAYER
        PLAYER -->|"재생 결과"| CTRL
    end

    subgraph PLATFORM["즉시 재생 플랫폼(100)"]
        direction TB
        VERIFY["VOD 재생 검증부(170)"]
        MANAGE["VOD 업체·콘텐츠 관리부(110)"]
        RECREG["추천 콘텐츠 등록부(130)"]
        STORE[("즉시 재생 콘텐츠 저장소(190)")]
        ADPROC["광고 처리부(140)"]
        GRANT["콘텐츠 재생권한 획득부(160)"]

        MANAGE <-->|"재생 검증 요청·결과"| VERIFY
        MANAGE -->|"검증된 업체·콘텐츠 등록정보"| STORE
        STORE --- RECREG
        STORE --- ADPROC
        STORE --- GRANT
    end

    MANAGE <-->|"업체·콘텐츠 등록 / 등록 결과"| VOD
    RECREG -->|"추천 콘텐츠 등록"| RECOMMEND
    RECOMMEND -->|"추천 콘텐츠 제공"| UI
    CTRL -->|"콘텐츠 정보"| ADPROC
    ADPROC -->|"광고 목록"| CTRL
    ADCHECK <-->|"광고 시청 확인"| ADPROC
    CTRL <-->|"재생권한 획득 요청·결과"| GRANT
    GRANT -->|"재생권한 요청"| VOD
    VOD -->|"한시적 재생권한"| GRANT
~~~

사용자는 사용자 단말(300)의 사용자 인터페이스(320)를 통해 콘텐츠를 선택한다. 사용자가 즉시 재생 플랫폼(100), 추천 시스템(400) 또는 VOD 업체 시스템(200)에 직접 재생 선택을 입력하는 관계는 형성되지 않는다.

### 3.3 구성요소별 역할

| 소속 | 도면부호 | 구성요소 | 역할 |
|---|---:|---|---|
| 플랫폼 | 110 | VOD 업체·콘텐츠 관리부 | 업체·콘텐츠·권한 요청정보를 등록받고, 검증을 지시하며, 등록상태를 관리한다. 등록용 외부 인터페이스를 포함할 수 있다. |
| 플랫폼 | 130 | 추천 콘텐츠 등록부 | 저장소에서 활성화된 즉시 재생 콘텐츠를 조회하여 추천 시스템에 등록한다. |
| 플랫폼 | 140 | 광고 처리부 | 선택 콘텐츠 정보를 이용하여 광고 목록을 제공하고, 단말의 광고 시청 확인부와 광고 완료를 확인한다. |
| 플랫폼 | 160 | 콘텐츠 재생권한 획득부 | 광고 시청 확인 후 저장소의 업체·콘텐츠 정보를 이용하여 VOD 업체 시스템에서 한시적 재생권한을 획득한다. |
| 플랫폼 | 170 | VOD 재생 검증부 | 등록된 권한 요청방법과 재생정보를 이용하여 콘텐츠의 실제 재생 가능성을 시험한다. |
| 플랫폼 | 190 | 즉시 재생 콘텐츠 저장소 | 검증된 업체·콘텐츠·권한 요청정보, 광고 연계정보와 등록상태를 저장한다. |
| VOD 업체 | 200 | VOD 업체 시스템 | 등록정보를 제공하고, 콘텐츠 재생권한을 발급하며, 콘텐츠를 제공한다. |
| 사용자 단말 | 310 | 즉시 재생 제어부 | 광고정보 획득, 광고 재생, 광고 확인, 권한 획득 및 콘텐츠 재생의 순서를 제어한다. |
| 사용자 단말 | 320 | 사용자 인터페이스 | 추천 콘텐츠를 표시하고 사용자의 즉시 재생 선택을 즉시 재생 제어부에 전달한다. |
| 사용자 단말 | 330 | 광고 시청 확인부 | 멀티미디어 재생기의 광고 재생결과를 바탕으로 광고 처리부와 시청 완료를 확인한다. |
| 사용자 단말 | 340 | 멀티미디어 재생기 | 즉시 재생 제어부의 제어에 따라 광고와 VOD 콘텐츠를 실제로 재생하고 결과를 반환한다. |
| 추천 시스템 | 400 | 추천 시스템 | 플랫폼이 등록한 즉시 재생 콘텐츠를 사용자 단말에 제공한다. |

멀티미디어 재생기(340)는 단말에 이미 구비된 일반적인 재생기일 수 있다. 본 발명의 즉시 재생 제어부(310)는 기존 재생기를 호출하고 재생 순서를 제어할 뿐, 스트리밍·DRM 또는 콘텐츠 보호방식 자체를 변경하지 않는다.

#### 광고 확인, 권한 획득과 콘텐츠 재생

~~~text
즉시 재생 제어부(310) ↔ 광고 처리부(140)

→ 즉시 재생 제어부(310)
→ 멀티미디어 재생기(340)에서 VOD 콘텐츠 재생
~~~

### 3.6 특허상 핵심 결합관계

1. VOD 업체 시스템이 업체·콘텐츠 정보와 콘텐츠 재생권한 요청방법을 플랫폼에 등록한다.
2. 플랫폼이 등록 콘텐츠의 권한 획득 및 실제 재생 가능성을 검증한다.
3. 검증된 콘텐츠만 즉시 재생 콘텐츠 저장소에 활성화되고 추천 시스템에 등록된다.
4. 사용자가 사용자 단말에서 콘텐츠를 선택하면 즉시 재생 제어부가 플랫폼의 광고 처리부로부터 해당 콘텐츠의 필수 광고 목록을 획득한다.
5. 즉시 재생 제어부가 멀티미디어 재생기로 광고를 재생하고, 단말의 광고 시청 확인부가 플랫폼의 광고 처리부와 광고 완료를 확인한다.
6. 즉시 재생 제어부는 광고 완료가 확인된 경우에만 콘텐츠 재생권한 획득부에 권한 획득을 요청한다.
7. 콘텐츠 재생권한 획득부는 저장소의 업체·콘텐츠 연결정보를 이용하여 해당 VOD 업체 시스템에서 한시적 재생권한을 획득한다.
8. 즉시 재생 제어부는 획득 권한을 이용하여 멀티미디어 재생기로 VOD 콘텐츠를 재생한다.
9. 콘텐츠 재생 중 추가 광고가 요구되는 경우 광고 확인과 다음 재생구간의 권한 획득을 반복할 수 있다.

## 4. VOD 업체·콘텐츠 등록, 검증 및 저장

### 4.1 VOD 업체·콘텐츠 관리부(110)의 역할

VOD 업체·콘텐츠 관리부(110)는 도면에서 VOD 업체 시스템(200), VOD 재생 검증부(170) 및 즉시 재생 콘텐츠 저장소(190)를 연결한다. 이 구성은 다음 기능을 수행한다.

- VOD 업체의 등록 요청 인증과 접수
- 업체별 콘텐츠 재생권한 서비스의 주소·호출방식·인증방식 등록
- 즉시 재생 대상 콘텐츠와 업체 콘텐츠 식별자의 등록
- 콘텐츠의 재생정보·제공조건·광고정책 참조정보 접수
- VOD 재생 검증부(170)에 콘텐츠별 시험 재생 요청
- 검증결과에 따른 활성·대기·거절 상태 결정
- 검증된 등록정보를 즉시 재생 콘텐츠 저장소(190)에 저장
- 등록 처리결과를 VOD 업체 시스템에 반환

등록용 통신 인터페이스는 VOD 업체·콘텐츠 관리부(110)의 일부로 구현될 수 있으므로 별도의 필수 구성부로 한정하지 않는다.

### 4.2 등록과 검증 절차

~~~mermaid
sequenceDiagram
    participant VOD as VOD 업체 시스템(200)
    participant MANAGE as VOD 업체·콘텐츠 관리부(110)
    participant VERIFY as VOD 재생 검증부(170)
    participant STORE as 즉시 재생 콘텐츠 저장소(190)
    participant RECREG as 추천 콘텐츠 등록부(130)
    participant RECOMMEND as 추천 시스템(400)

    VOD->>MANAGE: 업체·권한 서비스 등록정보
    MANAGE->>MANAGE: 형식·인증·호환성 확인
    MANAGE->>STORE: 업체 기준정보 저장
    MANAGE-->>VOD: 업체 등록결과

    VOD->>MANAGE: 콘텐츠·재생·권한 연결정보
    MANAGE->>VERIFY: 등록된 권한·재생정보와 콘텐츠 검증 요청
    VERIFY->>VERIFY: 등록정보로 검증용 권한 획득과 시험 재생
    VERIFY-->>MANAGE: 시험 재생결과
    alt 검증 성공
        MANAGE->>STORE: 콘텐츠 활성정보와 검증결과 저장
        STORE-->>RECREG: 추천 등록 가능한 콘텐츠
        RECREG->>RECOMMEND: 즉시 재생 콘텐츠 등록
        MANAGE-->>VOD: 콘텐츠 활성화 결과
    else 검증 실패
        MANAGE->>STORE: 검증실패 상태 저장
        MANAGE-->>VOD: 콘텐츠 비활성화 결과
    end
~~~

VOD 재생 검증부(170)가 사용하는 검증용 권한은 등록시험 전용이다. 실제 사용자의 광고 시청과 무관하며 사용자 재생에 재사용하지 않는다. 사용자 재생용 권한은 광고 시청 완료가 확인된 후 콘텐츠 재생권한 획득부(160)가 별도로 요청한다.

### 4.3 즉시 재생 콘텐츠 저장소(190)에 저장하는 데이터의 범위

저장소는 VOD 업체의 일반 사업정보 전체가 아니라 다음 즉시 재생 처리를 위해 필요한 정보만 저장한다.

- 업체 및 콘텐츠의 식별
- 업체별 콘텐츠 재생권한 요청
- 등록단계 시험 재생
- 검증된 콘텐츠의 추천 시스템 등록
- 선택 콘텐츠에 대응하는 필수 광고정보 연결
- 한시적 콘텐츠 재생권한 획득
- 광고·권한·재생기록의 업체 귀속과 수익배분

#### 업체 단위 데이터

| 데이터 | 사용 목적 |
|---|---|
| VOD 업체 식별자 | 콘텐츠 소유업체와 권한 요청대상 결정 |
| 콘텐츠 재생권한 서비스 식별자와 주소 | 콘텐츠 재생권한 획득부(160)의 호출대상 결정 |
| 서비스 호출방식 | 권한 요청의 전송방법 결정 |
| 서비스 인증방식 참조값 | 플랫폼과 업체 사이의 요청 인증방법 선택 |
| 권한 요청·응답 규격 버전 | 플랫폼 요청과 업체 응답의 호환성 확인 |
| 업체 응답 검증키 참조값 | 업체가 발급한 재생권한의 진위 확인 |
| 재생정보 제공주소 | 획득 권한으로 콘텐츠를 요청할 대상 결정 |
| 지원 스트리밍 방식 | 시험 재생과 단말 재생의 호환성 확인 |
| 지원 콘텐츠 보호 방식 | 기존 DRM 연동 가능성 확인 |
| 수익배분 정책 참조값 | 광고수익의 업체 귀속과 정산 |
| 등록 버전·활성상태 | 현재 사용할 업체 기준정보 결정 |

#### 콘텐츠 단위 데이터

| 데이터 | 사용 목적 |
|---|---|
| 업체 콘텐츠 식별자 | VOD 업체 내부의 권한 발급대상 지정 |
| 플랫폼 콘텐츠 식별자 | 추천·선택·광고·권한·재생기록 연결 |
| 재생 자산 식별자 | 실제 재생 대상 지정 |
| 제공지역·제공기간 | 현재 즉시 재생 가능 여부 확인 |
| 스트리밍·콘텐츠 보호 방식 | 시험 및 실제 재생방식 지정 |
| 즉시 재생 활성상태 | 추천 시스템 등록 가능 여부 결정 |
| 필수 광고정책 참조값 | 광고정보 또는 광고 목록 연결 |
| 한시적 재생권한 유효시간 | 업체에 요청할 권한의 최대 유효기간 지정 |
| 허용 재생범위 | 전체 콘텐츠 또는 특정 재생구간 지정 |
| VOD 업체 로그인 요구 여부 | 비로그인 즉시 재생 가능 여부 확인 |
| 재생 검증상태 | 시험 재생의 성공·실패와 유효성 확인 |
| 등록 버전·상태 | 현재 적용할 콘텐츠 기준정보 결정 |

다음 정보는 즉시 재생의 핵심 등록정보로 저장하지 않는다.

- 광고 소재의 내부 형식과 광고 선정 알고리즘
- 사용자의 VOD 업체 계정·구독·구매내역
- 장기 사용자 추적용 개인 식별정보
- 실제 영상파일 또는 장기 사용 가능한 재생주소
- DRM 키와 복호화된 콘텐츠
- 업체 서버 인증비밀의 원문
- 추천 알고리즘의 모델 또는 내부 점수

## 5. 추천 콘텐츠 등록과 사용자 선택

### 5.1 추천 콘텐츠 등록부(130)

추천 콘텐츠 등록부(130)는 즉시 재생 콘텐츠 저장소(190)에서 다음 조건을 충족하는 콘텐츠를 조회하여 추천 시스템(400)에 등록한다.

- 업체와 콘텐츠가 활성 상태임
- VOD 재생 검증결과가 성공이고 유효함
- 제공지역과 제공기간이 유효함
- VOD 업체 로그인이 필요하지 않음
- 필수 광고정책과 콘텐츠 재생권한 연결정보가 존재함

추천 콘텐츠 등록부는 추천 알고리즘을 수행하는 구성일 필요가 없다. 검증된 즉시 재생 콘텐츠를 추천 시스템이 사용할 수 있는 형식으로 등록하고, 비활성화 또는 검증 만료 시 등록을 해제하는 연결 구성이다.

### 5.2 추천 시스템(400)과 사용자 인터페이스(320)

추천 시스템(400)은 등록된 즉시 재생 콘텐츠를 사용자 단말(300)의 사용자 인터페이스(320)에 제공한다. 사용자는 사용자 인터페이스에서 콘텐츠를 선택하고, 사용자 인터페이스는 선택된 즉시 재생 콘텐츠 정보를 즉시 재생 제어부(310)에 전달한다.

사용자가 추천 시스템이나 즉시 재생 플랫폼에 직접 요청하는 것이 아니다. 사용자 입력의 직접 상대는 사용자 단말의 사용자 인터페이스이다.

선택정보는 다음 항목을 포함할 수 있다.

| 정보 | 의미 |
|---|---|
| 플랫폼 콘텐츠 식별자 | 선택된 콘텐츠를 플랫폼 저장정보와 연결 |
| 추천 등록 버전 | 선택 시 표시된 콘텐츠정보의 버전 |
| 익명 재생 세션 식별자 | 광고·권한·재생 처리를 연결 |
| 단말 지원정보 | 재생방식의 호환성 확인 |
| 현재 재생위치 | 추가 광고 이후 재생인 경우 다음 구간 결정 |

콘텐츠 추천의 점수계산, 개인화 방식 또는 화면 배열 자체는 본 발명의 중심이 아니다. 권리화 관점에서는 검증된 즉시 재생 콘텐츠만 추천 시스템에 등록되고, 선택된 콘텐츠 식별자가 광고 처리와 권한 획득에 동일하게 사용되는 관계가 중요하다.

## 6. 사용자 단말의 즉시 재생 구조

### 6.1 사용자 인터페이스(320)

사용자 인터페이스(320)는 추천 시스템(400)으로부터 즉시 재생 콘텐츠를 제공받아 표시하고, 사용자의 선택을 즉시 재생 제어부(310)에 전달한다. 사용자 인터페이스는 광고 완료를 판단하거나 콘텐츠 재생권한을 획득하지 않는다.

### 6.2 즉시 재생 제어부(310)

즉시 재생 제어부(310)는 단말에서 다음 실행 순서를 제어하는 중심 구성이다.

1. 사용자 인터페이스에서 선택 콘텐츠 정보를 수신한다.
2. 선택 콘텐츠 정보를 광고 처리부(140)에 전달하고 광고 목록을 수신한다.
3. 멀티미디어 재생기(340)에 광고 재생을 지시한다.
4. 멀티미디어 재생기의 광고 재생결과를 광고 시청 확인부(330)에 제공한다.
5. 광고 시청 확인부가 광고 처리부와 확인한 광고 시청 확인정보를 수신한다.
6. 광고 완료가 확인된 경우에만 콘텐츠 재생권한 획득부(160)에 권한 획득을 요청한다.
7. 획득한 한시적 콘텐츠 재생권한을 멀티미디어 재생기에 제공한다.
8. VOD 콘텐츠의 재생 시작·일시정지·계속 또는 종료를 제어한다.

즉시 재생 제어부는 광고 시청 확인 전에는 사용자 재생용 콘텐츠 재생권한을 요청하지 않으며, 유효한 권한을 획득하기 전에는 멀티미디어 재생기에 VOD 콘텐츠 재생을 지시하지 않는다.

### 6.3 광고 시청 확인부(330)

광고 시청 확인부(330)는 사용자 단말에 배치된다. 즉시 재생 제어부로부터 광고 재생결과와 확인 대상정보를 수신하고, 플랫폼의 광고 처리부(140)와 통신하여 해당 광고 요구가 완료되었는지를 확인한다.

광고 시청 확인부는 단말의 재생결과만을 자체 신뢰하여 재생권한을 허용하지 않는다. 플랫폼의 광고 처리부가 보유한 광고 요청정보 또는 광고 목록과 대조한 확인 결과를 즉시 재생 제어부에 반환한다. 이에 따라 단말에서 임의로 생성한 광고 완료 상태가 바로 콘텐츠 권한 요청으로 이어지는 것을 제한할 수 있다.

### 6.4 멀티미디어 재생기(340)

멀티미디어 재생기(340)는 즉시 재생 제어부의 재생명령에 따라 광고 또는 VOD 콘텐츠를 실제 재생하고 재생결과를 반환한다. 다음과 같은 기존 단말 기능을 그대로 사용할 수 있다.

- 광고 및 콘텐츠 데이터의 디코딩과 출력
- 스트리밍 재생
- 기존 DRM 처리
- 재생 시작·일시정지·계속·종료
- 재생위치와 완료상태 반환

멀티미디어 재생기는 광고 시청 완료의 최종 확인주체도 아니고 VOD 콘텐츠 재생권한의 발급주체도 아니다.

### 6.5 즉시 재생 세션

즉시 재생 제어부는 다음 정보를 포함하는 짧은 수명의 세션을 유지할 수 있다.

- 익명 재생 세션 식별자
- 플랫폼·업체 콘텐츠 식별자
- 콘텐츠 등록 버전
- 광고 요청 또는 광고 목록 식별자
- 광고 요구시점 식별자
- 광고 재생결과와 광고 시청 확인정보
- 콘텐츠 재생권한 요청 식별자
- VOD 업체가 발급한 권한의 식별자와 만료시각
- 현재 재생구간과 재생위치
- 현재 상태: 콘텐츠선택·광고대기·광고재생·광고확인·권한대기·콘텐츠재생·일시정지·완료·실패

---

## 7. 광고 처리부(140)와 광고 시청 확인부(330)

### 7.1 역할의 분리

| 구성 | 역할 |
|---|---|
| 광고 처리부(140) | 선택 콘텐츠 정보에 대응하는 광고 목록을 제공하고, 광고 요청정보와 단말의 확인 요청을 대조하여 광고 완료 확인결과를 제공 |
| 광고 시청 확인부(330) | 멀티미디어 재생기의 광고 재생결과를 수신하고 광고 처리부와 광고 완료를 확인하여 즉시 재생 제어부에 결과 제공 |
| 즉시 재생 제어부(310) | 확인결과가 성공인 경우에만 콘텐츠 재생권한 획득을 요청 |

광고의 종류, 표시형식, 과금방식, 광고 수 또는 광고 선정 알고리즘은 본 발명의 중심이 아니다. 선택 콘텐츠와 연결된 필수 광고가 재생되고, 단말의 광고 시청 확인부와 플랫폼의 광고 처리부 사이에서 완료가 확인된 후에만 콘텐츠 재생권한 획득이 진행된다는 제어관계가 핵심이다.

### 7.2 광고 확인 절차

1. 즉시 재생 제어부가 콘텐츠 식별자와 세션 식별자를 광고 처리부에 전달한다.
2. 광고 처리부가 해당 콘텐츠에 대응하는 광고 목록 또는 광고 요구정보를 반환한다.
3. 즉시 재생 제어부가 멀티미디어 재생기에 광고 재생을 지시한다.
4. 멀티미디어 재생기가 재생결과를 즉시 재생 제어부에 반환한다.
5. 즉시 재생 제어부가 광고 재생결과를 광고 시청 확인부에 제공한다.
6. 광고 시청 확인부가 광고 처리부에 광고 완료 확인을 요청한다.
7. 광고 처리부가 원래 제공한 광고 목록, 콘텐츠, 세션 및 광고 요구시점과 대조한다.
8. 광고 시청 확인부가 확인정보를 수신하여 즉시 재생 제어부에 반환한다.
9. 즉시 재생 제어부는 확인 성공인 경우에만 콘텐츠 재생권한 획득부를 호출한다.

### 7.3 광고 시청 확인정보의 예

다음 형식은 하나의 실시예이다. 확인정보는 토큰, 서명된 메시지 또는 서버 측 세션상태로 구현될 수 있다.

~~~json
{
  "confirmationType": "instantplay.ad-confirmation/1.0",
  "adConfirmationId": "ad-confirmation-01J...",
  "anonymousSessionId": "session-01J...",
  "platformContentId": "platform-title-7788",
  "adCheckpointId": "checkpoint-initial",
  "adRequestId": "ad-request-01J...",
  "adListVersion": 3,
  "completionStatus": "completed",
  "confirmedAt": "2026-07-19T01:01:24Z",
  "nonce": "ONE_TIME_CONFIRMATION_NONCE",
  "confirmationProof": "SIGNED_AD_CONFIRMATION"
}
~~~

| 필드 | 의미 |
|---|---|
| adConfirmationId | 광고 시청 확인정보의 식별자 |
| anonymousSessionId | 광고와 권한 획득을 연결하는 즉시 재생 세션 |
| platformContentId | 광고 완료가 적용되는 콘텐츠 |
| adCheckpointId | 최초 광고 또는 콘텐츠 재생 중 특정 광고 요구시점 |
| adRequestId | 광고 처리부가 생성한 광고 요청과의 연결키 |
| adListVersion | 실제 제공된 광고 목록의 버전 |
| completionStatus | 완료·미완료·실패 상태 |
| confirmedAt | 광고 완료 확인시각 |
| nonce | 확인정보의 반복 사용 방지값 |
| confirmationProof | 광고 처리부와 확인된 결과임을 나타내는 무결성 정보 |

광고 시청 확인정보는 VOD 콘텐츠 재생권한 자체가 아니다. 콘텐츠 재생권한 획득부(160)를 호출할 수 있게 하는 선행조건 또는 그 선행조건의 증명정보이다.

### 7.4 최초 광고와 추가 광고

필수 광고는 콘텐츠 최초 재생 전에 요구될 수 있고 콘텐츠 재생 중 하나 이상의 광고 요구시점에서 다시 요구될 수 있다. 추가 광고가 요구되면 즉시 재생 제어부는 멀티미디어 재생기의 VOD 재생을 일시정지하고, 광고 재생과 광고 확인을 다시 수행한다. 확인 후에는 다음 재생구간에 대응하는 새로운 한시적 재생권한을 획득할 수 있다.

## 8. 콘텐츠 재생권한 획득부(160)

### 8.1 역할과 권한 발급주체

콘텐츠 재생권한 획득부(160)는 즉시 재생 플랫폼(100)에 배치된다. 이 구성은 자체적으로 VOD 콘텐츠 재생권한을 생성하지 않는다.

1. 즉시 재생 제어부(310)로부터 콘텐츠 정보와 광고 시청 확인정보를 포함하는 권한 획득 요청을 수신한다.
2. 광고 시청 확인정보가 요청 콘텐츠·세션·광고 요구시점과 일치하는지 확인한다.
3. 즉시 재생 콘텐츠 저장소(190)에서 해당 VOD 업체, 업체 콘텐츠 식별자, 권한 서비스 및 재생정보를 조회한다.
4. 등록된 방식으로 VOD 업체 시스템(200)에 콘텐츠 재생권한을 요청한다.
5. VOD 업체가 발급한 한시적 콘텐츠 재생권한을 수신하고 검증한다.
6. 획득한 권한을 즉시 재생 제어부에 반환한다.

광고 완료가 확인되지 않았거나 확인정보와 콘텐츠·세션이 일치하지 않으면 VOD 업체 시스템에 사용자 재생용 권한을 요청하지 않는다.

### 8.2 저장소 조회정보

- 활성 VOD 업체 식별자
- 업체의 콘텐츠 재생권한 서비스 주소
- 서비스 호출방식과 인증방식 참조값
- 권한 요청·응답 규격 버전
- 업체 응답 검증키 참조값
- 업체·플랫폼 콘텐츠 식별자
- 재생 자산 식별자
- 콘텐츠 제공지역과 제공기간
- 허용 재생범위와 권한 유효시간
- 콘텐츠 등록 버전과 재생 검증상태
- 광고정책 참조값과 현재 광고 요구시점

### 8.3 VOD 업체에 전송하는 권한 요청의 예

~~~json
{
  "requestType": "instantplay.grant-request/1.0",
  "requestId": "grant-request-01J...",
  "providerId": "com.example.vod",
  "providerContentId": "movie-12345",
  "playbackResourceId": "asset-5f91a2",
  "anonymousSessionId": "session-01J...",
  "adConfirmation": {
    "adConfirmationId": "ad-confirmation-01J...",
    "adCheckpointId": "checkpoint-initial",
    "confirmedAt": "2026-07-19T01:01:24Z",
    "confirmationProof": "SIGNED_AD_CONFIRMATION"
  },
  "requestedAccess": {
    "playbackScope": "playback-segment",
    "segmentIndex": 1,
    "maximumTtlSec": 120
  },
  "registrationVersion": 12,
  "nonce": "ONE_TIME_REQUEST_NONCE",
  "platformSignature": "SIGNED_PLATFORM_REQUEST"
}
~~~

| 필드 | 의미 |
|---|---|
| requestId | 플랫폼 요청과 업체 응답을 연결하는 식별자 |
| providerId | 권한을 발급할 VOD 업체 |
| providerContentId | 업체 내부의 권한 대상 콘텐츠 |
| playbackResourceId | 실제 재생 자산 |
| anonymousSessionId | 광고 확인·권한 획득·콘텐츠 재생을 연결하는 세션 |
| adConfirmation | 권한 획득의 선행조건이 된 광고 시청 확인정보 |
| requestedAccess | 요청하는 재생범위·구간·유효시간 |
| registrationVersion | 적용한 콘텐츠 등록정보 버전 |
| nonce | 권한 요청의 반복 사용 방지값 |
| platformSignature | 등록된 플랫폼의 요청임을 업체가 확인하는 값 |

광고 시청 확인정보를 VOD 업체에 그대로 전달할지는 업체별 연동방식에 따라 달라질 수 있다. 콘텐츠 재생권한 획득부가 플랫폼 내부에서 확인한 후 그 확인결과만 권한 요청에 반영하는 방식도 포함할 수 있다.

### 8.4 VOD 업체가 반환하는 권한의 예

~~~json
{
  "responseType": "instantplay.grant-response/1.0",
  "requestId": "grant-request-01J...",
  "grantId": "vendor-grant-01J...",
  "issuer": "com.example.vod",
  "providerContentId": "movie-12345",
  "playbackResourceId": "asset-5f91a2",
  "anonymousSessionId": "session-01J...",
  "adCheckpointId": "checkpoint-initial",
  "permittedPlayback": {
    "scope": "playback-segment",
    "segmentIndex": 1
  },
  "playbackAccessToken": "SHORT_LIVED_VENDOR_TOKEN",
  "issuedAt": "2026-07-19T01:01:25Z",
  "expiresAt": "2026-07-19T01:03:25Z",
  "vendorSignature": "SIGNED_VENDOR_GRANT"
}
~~~

콘텐츠 재생권한 획득부는 업체 응답의 요청 식별자, 발급주체, 콘텐츠, 세션, 재생범위, 만료시각과 서명을 확인한다. 검증에 성공하면 획득 권한을 즉시 재생 제어부(310)에 전달한다.

### 8.5 권한의 범위와 반복 획득

한시적 콘텐츠 재생권한은 다음 중 하나 이상으로 제한될 수 있다.

- 특정 콘텐츠
- 특정 익명 재생 세션
- 특정 재생구간
- 짧은 유효시간
- 단일 재생 또는 제한된 재생횟수
- 특정 단말 또는 재생 요청

최초 광고 확인 후에는 콘텐츠 시작구간의 권한을 획득하고, 추가 광고 확인 후에는 다음 재생구간의 권한을 새로 획득할 수 있다. 하나의 광고 시청 확인정보를 서로 다른 콘텐츠·세션·광고 요구시점의 권한 획득에 재사용하지 않도록 제한할 수 있다.

## 9. VOD 재생 검증부(170)

### 9.1 등록단계 검증

VOD 재생 검증부(170)는 업체·콘텐츠 등록단계에서 동작한다. VOD 업체·콘텐츠 관리부(110)의 요청에 따라 등록된 권한 서비스에 검증용 권한을 요청하고, 획득 권한으로 콘텐츠가 실제 재생되는지를 시험한다.

### 9.2 검증 절차

1. 등록된 업체 권한 서비스 주소와 인증방식으로 검증용 권한을 요청한다.
2. VOD 업체 시스템이 검증용 한시적 권한을 반환하는지 확인한다.
3. 검증용 권한으로 등록 콘텐츠의 재생정보를 요청한다.
4. 재생목록 또는 콘텐츠 주소에 접근 가능한지 확인한다.
5. 등록된 스트리밍·콘텐츠 보호 방식으로 재생을 시작할 수 있는지 확인한다.
6. 단계별 결과를 VOD 업체·콘텐츠 관리부에 반환한다.

### 9.3 검증결과의 예

~~~json
{
  "verificationType": "instantplay.registration-validation/2.0",
  "verificationId": "verification-01J...",
  "providerId": "com.example.vod",
  "providerContentId": "movie-12345",
  "registrationVersion": 12,
  "checks": {
    "temporaryGrantIssued": "passed",
    "playbackInfoResolved": "passed",
    "manifestAccessible": "passed",
    "playbackStarted": "passed"
  },
  "status": "passed",
  "verifiedAt": "2026-07-19T00:00:00Z",
  "validUntil": "2026-08-19T00:00:00Z",
  "evidenceHash": "sha256-..."
}
~~~

검증결과가 성공인 경우에만 콘텐츠를 활성화한다. 권한 서비스 주소, 재생 자산, 스트리밍 방식 또는 콘텐츠 보호 방식이 변경되면 기존 검증결과를 무효화하고 다시 검증할 수 있다.

## 10. 전체 동작 흐름

### 10.1 등록·검증·추천 흐름

등록·검증·추천의 전체 순서는 4.2의 흐름과 같다. 중요한 제어조건은 검증되지 않은 콘텐츠가 즉시 재생 콘텐츠 저장소의 활성 콘텐츠가 되거나 추천 시스템에 등록되지 않는다는 것이다.

### 10.2 사용자의 즉시 재생 흐름

~~~mermaid
sequenceDiagram
    autonumber
    actor USER as 사용자
    participant UI as 사용자 인터페이스(320)
    participant CTRL as 즉시 재생 제어부(310)
    participant PLAYER as 멀티미디어 재생기(340)
    participant ADCHECK as 광고 시청 확인부(330)
    participant ADPROC as 광고 처리부(140)
    participant STORE as 즉시 재생 콘텐츠 저장소(190)
    participant GRANT as 콘텐츠 재생권한 획득부(160)
    participant VOD as VOD 업체 시스템(200)

    USER->>UI: 사용자 단말에서 즉시 재생 콘텐츠 선택
    UI->>CTRL: 선택 콘텐츠 정보
    CTRL->>ADPROC: 콘텐츠·세션 정보
    ADPROC->>STORE: 콘텐츠·광고 연결정보 조회
    STORE-->>ADPROC: 필수 광고정책
    ADPROC-->>CTRL: 광고 목록
    CTRL->>PLAYER: 광고 재생명령과 광고정보
    PLAYER-->>USER: 사용자 단말에서 광고 출력
    PLAYER-->>CTRL: 광고 재생결과
    CTRL->>ADCHECK: 재생결과와 확인 대상정보
    ADCHECK->>ADPROC: 광고 완료 확인 요청
    ADPROC-->>ADCHECK: 광고 시청 확인정보
    ADCHECK-->>CTRL: 광고 완료 확인결과

    alt 광고 완료 확인
        CTRL->>GRANT: 콘텐츠·세션·광고 확인정보
        GRANT->>STORE: 업체·콘텐츠·권한 서비스 조회
        STORE-->>GRANT: 검증된 권한 요청정보
        GRANT->>VOD: 한시적 콘텐츠 재생권한 요청
        VOD-->>GRANT: VOD 업체가 발급한 재생권한
        GRANT-->>CTRL: 획득한 콘텐츠 재생권한
        CTRL->>PLAYER: 권한과 콘텐츠 재생명령
        PLAYER->>VOD: 권한을 제시하여 콘텐츠 요청
        VOD-->>PLAYER: VOD 콘텐츠
        PLAYER-->>USER: 사용자 단말에서 콘텐츠 출력
        PLAYER-->>CTRL: 콘텐츠 재생결과
    else 광고 미완료 또는 확인 실패
        CTRL->>CTRL: 권한 획득 요청 차단
        CTRL->>PLAYER: 콘텐츠 재생 금지
    end

    loop 하나 이상의 추가 광고 요구시점
        CTRL->>PLAYER: VOD 재생 일시정지
        CTRL->>ADPROC: 다음 광고 목록 요청
        ADPROC-->>CTRL: 다음 광고 목록
        CTRL->>PLAYER: 광고 재생
        PLAYER-->>CTRL: 광고 재생결과
        CTRL->>ADCHECK: 광고 확인 요청
        ADCHECK->>ADPROC: 광고 완료 대조
        ADPROC-->>ADCHECK: 다음 광고 시청 확인정보
        ADCHECK-->>CTRL: 완료 확인결과
        alt 추가 광고 완료 확인
            CTRL->>GRANT: 다음 재생구간의 권한 획득 요청
            GRANT->>VOD: 다음 구간의 한시적 권한 요청
            VOD-->>GRANT: 다음 구간의 콘텐츠 재생권한
            GRANT-->>CTRL: 획득 권한
            CTRL->>PLAYER: 콘텐츠 재생 계속
        else 추가 광고 미완료 또는 확인 실패
            CTRL->>CTRL: 다음 권한 획득 요청 차단
            CTRL->>PLAYER: 콘텐츠 재생 일시정지 또는 종료 유지
        end
    end
~~~

이 흐름에서 사용자의 직접 입력은 사용자 인터페이스(320)에만 전달된다. 광고 완료 확인은 단말의 광고 시청 확인부(330)와 플랫폼의 광고 처리부(140)가 수행하고, 재생권한은 VOD 업체 시스템(200)이 발급하며, 실제 광고·콘텐츠 출력은 멀티미디어 재생기(340)가 수행한다.

## 11. 보안, 실패 처리 및 복수 업체 지원

### 11.1 광고 시청 확인정보 보호

- 광고 확인정보에는 세션, 콘텐츠, 광고 요구시점, 광고 요청 식별자와 만료정보를 결합할 수 있다.
- 광고 처리부는 자신이 제공한 광고 목록과 단말의 확인 요청을 대조할 수 있다.
- 확인정보에 일회성 값, 서명 또는 무결성 정보를 포함할 수 있다.
- 하나의 광고 확인정보를 둘 이상의 권한 획득 요청에 사용하는 것을 차단할 수 있다.
- 광고 시청 확인부는 단말이 임의 생성한 완료상태만으로 권한 획득 성공을 결정하지 않는다.

### 11.2 권한 요청과 응답 보호

- 플랫폼과 VOD 업체 시스템은 상호 인증된 통신을 사용할 수 있다.
- 권한 요청에 콘텐츠, 세션, 광고 요구시점, 요청범위, 만료 및 중복방지값을 결합할 수 있다.
- VOD 업체는 플랫폼의 인증과 요청서명을 확인한 후 권한을 발급할 수 있다.
- 콘텐츠 재생권한 획득부는 업체 응답의 발급주체, 콘텐츠, 세션, 범위, 유효시간과 서명을 검증할 수 있다.
- 실제 인증비밀 원문 대신 안전한 참조값을 저장소에 보관할 수 있다.

기존 VOD 콘텐츠의 DRM, 암호화, 복호화와 보호 출력은 VOD 업체와 사용자 단말의 기존 기술을 사용한다. 본 발명은 새로운 하드웨어 보안영역을 필수로 요구하지 않는다.

복구 요청은 새로운 광고 시청 완료를 임의로 생성하지 않는다. 유효한 광고 확인정보와 동일 세션·콘텐츠·광고 요구시점에 결합된 권한 요청만 제한적으로 재시도할 수 있다.

### 11.4 복수 VOD 업체 지원

- 업체별 권한 서비스 주소, 인증방식, 요청·응답 규격과 검증키를 분리한다.
- 하나의 콘텐츠는 하나의 등록 업체와 권한 서비스를 명시적으로 참조한다.
- 다른 업체의 콘텐츠 식별자, 광고 확인정보 또는 권한 응답을 재사용하지 못하게 한다.
- 업체별 등록정보와 콘텐츠정보의 버전을 독립적으로 관리한다.
- 업체별 호출한도, 장애 차단과 수익배분 정책을 적용할 수 있다.

### 11.5 개인정보 최소화

- VOD 업체 사용자 계정을 생성하거나 요구하지 않는다.
- 영구 사용자 식별자 대신 짧은 수명의 익명 재생 세션을 사용할 수 있다.
- 광고 확인, 권한 요청 및 재생기록에는 기술적 대사와 수익배분에 필요한 최소 정보만 저장한다.
- 보존기간이 지난 세션정보는 삭제하거나 비식별화할 수 있다.

## 12. 광고수익과 VOD 업체 배분

광고수익 배분은 본 발명의 핵심 재생 제어와 결합될 수 있는 후속 처리이다. 도면에는 별도의 정산 구성부가 표시되어 있지 않으므로 필수 독립 구성으로 한정하지 않는다.

다음 사건을 동일한 익명 재생 세션, 콘텐츠 및 업체에 연결할 수 있다.

- 추천 콘텐츠 선택
- 광고 목록 요청과 광고 재생결과
- 광고 시청 확인정보
- VOD 업체에 대한 콘텐츠 재생권한 요청
- VOD 업체가 발급한 권한
- 멀티미디어 재생기의 콘텐츠 재생 시작·계속·완료결과
- 광고수익과 VOD 업체 배분액

광고 완료만 존재하고 VOD 업체가 권한 발급을 거절한 경우, 또는 권한은 획득했으나 실제 콘텐츠 재생이 실패한 경우에는 계약에 따라 배분을 보류할 수 있다. 수익배분 계산방법 자체보다 광고 확인·권한 획득·실제 재생을 동일 세션과 콘텐츠로 연결하는 기록구조가 기술적 근거가 된다.

## 13. 특허 청구항의 핵심 요소

### 13.1 핵심 권리화 명제

> 사용자 단말의 즉시 재생 제어부가 선택 콘텐츠에 대응하는 광고를 멀티미디어 재생기로 재생하고, 단말의 광고 시청 확인부가 플랫폼의 광고 처리부와 광고 완료를 확인한 경우에만 플랫폼의 콘텐츠 재생권한 획득부를 호출하며, 콘텐츠 재생권한 획득부가 등록된 업체·콘텐츠 연결정보를 이용하여 VOD 업체가 발급한 한시적 재생권한을 획득해 단말의 콘텐츠 재생에 제공하는 구조.

이 명제는 다음 세 가지 주체를 분리하면서도 하나의 조건부 제어사슬로 결합한다.

1. 광고 완료 확인 주체: 사용자 단말의 광고 시청 확인부와 플랫폼의 광고 처리부
2. 콘텐츠 재생권한 획득 주체: 즉시 재생 플랫폼의 콘텐츠 재생권한 획득부
3. 콘텐츠 재생권한 발급 주체: VOD 업체 시스템

### 13.2 독립 청구항에 우선 포함할 요소

| 우선도 | 청구 요소 | 포함 이유 |
|---|---|---|
| 필수 | 사용자 단말에서 콘텐츠 선택정보를 수신하는 즉시 재생 제어부 | 사용자와 플랫폼의 직접 연결이 아니라 단말 제어구조임을 특정 |
| 필수 | 콘텐츠 정보에 대응하는 필수 광고정보를 제공하는 플랫폼 광고 처리부 | 콘텐츠 선택과 광고 요구의 연결을 특정 |
| 필수 | 광고를 실제 재생하는 단말의 멀티미디어 재생기 | 광고·VOD가 같은 단말 실행경로에서 제어됨을 특정 |
| 필수 | 광고 처리부와 광고 완료를 확인하는 단말의 광고 시청 확인부 | 단말 자체판단이 아닌 단말·플랫폼 상호확인을 특정 |
| 필수 | 광고 완료 확인 후에만 권한 획득을 요청하는 즉시 재생 제어부 | 광고 시청이 콘텐츠 이용자격의 선행조건임을 특정 |
| 필수 | 등록된 업체·콘텐츠 연결정보를 조회하는 콘텐츠 재생권한 획득부 | 복수 VOD 업체의 이종 권한 서비스 연동을 특정 |
| 필수 | VOD 업체 시스템에 권한을 요청하여 업체 발급 권한을 획득하는 구성 | 플랫폼이 권한 발급자가 아니라는 경계를 특정 |
| 필수 | 획득 권한을 멀티미디어 재생기에 제공하여 콘텐츠를 재생하는 구성 | 광고 확인부터 실제 VOD 재생까지 기술적 결과 연결 |
| 강화 | 업체·콘텐츠 등록정보와 권한 요청방법을 저장하는 저장소 | 사전 등록 기반 연동을 특정 |
| 강화 | 등록 콘텐츠의 실제 재생 가능성을 검증하는 VOD 재생 검증부 | 검증되지 않은 업체·콘텐츠 차단이라는 차별점 강화 |
| 강화 | 검증된 콘텐츠만 추천 시스템에 등록하는 추천 콘텐츠 등록부 | 등록·검증·노출의 폐쇄된 제어사슬 형성 |

가장 넓은 독립 청구항은 광고 확인 후 VOD 업체 권한을 획득하는 재생 제어사슬에 집중하고, 등록 검증과 추천 등록은 별도 독립항 또는 종속항으로 구성할 수 있다. 반대로 선행기술과의 구별이 더 필요하면 등록·검증·추천 등록까지 하나의 결합으로 포함하는 협의의 독립항을 준비할 수 있다.

상기 즉시 재생 제어부의 제어에 따라 상기 필수 광고정보에 대응하는 광고를 재생하고 광고 재생결과를 반환하는 상기 사용자 단말의 멀티미디어 재생기;

상기 광고 재생결과를 이용하여 상기 광고 처리부와 상기 광고의 시청 완료를 확인하고 광고 시청 확인정보를 상기 즉시 재생 제어부에 제공하는 상기 사용자 단말의 광고 시청 확인부;

VOD 업체별 업체정보, 콘텐츠정보 및 콘텐츠 재생권한 요청정보가 저장된 즉시 재생 콘텐츠 저장소; 및

상기 광고 시청 확인정보가 존재하는 경우 상기 저장소에서 상기 선택된 콘텐츠에 대응하는 VOD 업체정보, 업체 콘텐츠 식별자 및 콘텐츠 재생권한 요청정보를 조회하고, 상기 조회된 정보에 따라 VOD 업체 시스템에 한시적 콘텐츠 재생권한을 요청하며, 상기 VOD 업체 시스템이 발급한 콘텐츠 재생권한을 획득하는 콘텐츠 재생권한 획득부를 포함하고,

상기 즉시 재생 제어부는 상기 광고 시청 확인부로부터 상기 광고 시청 확인정보를 수신한 경우에만 상기 콘텐츠 재생권한 획득부에 상기 콘텐츠 재생권한의 획득을 요청하고, 상기 콘텐츠 재생권한 획득부가 획득한 상기 콘텐츠 재생권한을 상기 멀티미디어 재생기에 제공하여 상기 VOD 업체 시스템의 콘텐츠를 재생시키는 즉시 재생 시스템.
~~~

이 청구항의 중심은 “광고 재생” 자체가 아니라 `단말 광고 시청 확인부 ↔ 플랫폼 광고 처리부의 완료 확인 → 단말 제어부의 조건부 권한 획득 요청 → 플랫폼 권한 획득부의 VOD 업체 권한 획득 → 단말 재생`의 순차적 의존관계이다.

### 13.4 독립 플랫폼 청구항 후보: 등록·검증·추천·권한 획득

~~~text
복수의 VOD 업체 시스템과 연동하는 즉시 재생 플랫폼에 있어서,

VOD 업체 시스템에 업체 및 콘텐츠의 등록방법을 제공하고, 상기 VOD 업체 시스템으로부터 업체정보, 콘텐츠정보, 콘텐츠 재생정보 및 콘텐츠 재생권한 요청정보를 등록받는 등록 인터페이스를 포함하는 VOD 업체·콘텐츠 관리부;

상기 콘텐츠 재생권한 요청정보와 상기 콘텐츠 재생정보를 이용하여 상기 VOD 업체 시스템으로부터 검증용 콘텐츠 재생권한을 획득하고 등록 콘텐츠의 실제 재생 가능성을 시험하는 VOD 재생 검증부;

상기 VOD 재생 검증부의 검증에 성공한 콘텐츠에 대응하는 상기 업체정보, 콘텐츠정보 및 콘텐츠 재생권한 요청정보를 활성상태로 저장하는 즉시 재생 콘텐츠 저장소;

상기 활성상태의 콘텐츠를 추천 시스템에 등록하는 추천 콘텐츠 등록부;

사용자 단말에서 선택된 콘텐츠에 대응하는 필수 광고정보를 제공하고, 상기 사용자 단말의 광고 시청 확인부와 광고 시청 완료를 확인하는 광고 처리부; 및

상기 광고 시청 완료가 확인된 경우 상기 저장소에서 상기 선택된 콘텐츠에 대응하는 업체정보와 콘텐츠 재생권한 요청정보를 조회하고, 해당 VOD 업체 시스템에 한시적 콘텐츠 재생권한을 요청하여 상기 VOD 업체 시스템이 발급한 콘텐츠 재생권한을 획득하는 콘텐츠 재생권한 획득부를 포함하는 즉시 재생 플랫폼.
~~~

이 청구항은 복수 업체를 하나의 플랫폼에 등록시키는 것만이 아니라, 실제 권한 획득과 재생을 검증한 콘텐츠만 추천 대상으로 전환하고, 사용 시점에는 광고 완료를 조건으로 다시 업체 권한을 획득하는 이중 검증구조를 강조한다.

### 13.5 독립 사용자 단말 청구항 후보

~~~text
즉시 재생 사용자 단말에 있어서,

추천 시스템으로부터 제공된 즉시 재생 콘텐츠를 표시하고 사용자의 콘텐츠 선택을 수신하는 사용자 인터페이스;

광고 또는 VOD 콘텐츠를 재생하고 재생결과를 반환하는 멀티미디어 재생기;

즉시 재생 플랫폼의 광고 처리부가 제공한 필수 광고정보에 따라 상기 멀티미디어 재생기에서 재생된 광고의 재생결과를 이용하여 상기 광고 처리부와 광고 시청 완료를 확인하는 광고 시청 확인부; 및

상기 사용자 인터페이스가 전달한 선택 콘텐츠 정보를 상기 광고 처리부에 제공하고, 상기 필수 광고정보를 상기 멀티미디어 재생기에 제공하며, 상기 광고 시청 확인부가 광고 시청 완료를 확인한 경우에만 상기 즉시 재생 플랫폼에 상기 선택 콘텐츠의 콘텐츠 재생권한 획득을 요청하고, 상기 즉시 재생 플랫폼이 VOD 업체 시스템으로부터 획득한 콘텐츠 재생권한을 상기 멀티미디어 재생기에 제공하여 상기 VOD 콘텐츠를 재생시키는 즉시 재생 제어부를 포함하는 사용자 단말.
~~~

단말 청구항은 플랫폼 서버 구현과 별개로, 사용자 단말 내부의 사용자 인터페이스·제어부·광고 확인부·재생기 결합을 보호하는 데 사용할 수 있다.

### 13.6 독립 방법 청구항 후보

~~~text
사용자 단말의 사용자 인터페이스가 사용자의 즉시 재생 콘텐츠 선택을 수신하는 단계;

상기 사용자 단말의 즉시 재생 제어부가 선택 콘텐츠 정보를 즉시 재생 플랫폼의 광고 처리부에 제공하고, 상기 광고 처리부로부터 상기 선택 콘텐츠에 대응하는 필수 광고정보를 수신하는 단계;

상기 즉시 재생 제어부가 상기 필수 광고정보를 상기 사용자 단말의 멀티미디어 재생기에 제공하여 광고를 재생시키는 단계;

상기 사용자 단말의 광고 시청 확인부가 상기 멀티미디어 재생기의 광고 재생결과를 이용하여 상기 광고 처리부와 광고 시청 완료를 확인하는 단계;

상기 광고 시청 완료가 확인된 경우에만 상기 즉시 재생 제어부가 상기 즉시 재생 플랫폼의 콘텐츠 재생권한 획득부에 콘텐츠 재생권한의 획득을 요청하는 단계;

상기 콘텐츠 재생권한 획득부가 즉시 재생 콘텐츠 저장소에서 상기 선택 콘텐츠에 대응하는 VOD 업체정보, 업체 콘텐츠 식별자 및 콘텐츠 재생권한 요청정보를 조회하는 단계;

상기 콘텐츠 재생권한 획득부가 상기 조회된 정보에 따라 VOD 업체 시스템에 한시적 콘텐츠 재생권한을 요청하고, 상기 VOD 업체 시스템이 발급한 콘텐츠 재생권한을 획득하는 단계; 및

상기 즉시 재생 제어부가 상기 획득한 콘텐츠 재생권한을 상기 멀티미디어 재생기에 제공하여 상기 VOD 업체 시스템의 콘텐츠를 재생시키는 단계를 포함하는 즉시 재생 방법.
~~~

### 13.7 결합형 독립항 후보

선행기술과의 구별을 강화하기 위해 다음 세 개의 제어사슬을 하나의 독립항에 결합할 수 있다.

1. **등록·검증 사슬:** 업체·콘텐츠 등록 → 검증용 권한 획득 → 실제 재생 검증 → 활성 콘텐츠 저장
2. **추천·선택 사슬:** 활성 콘텐츠 → 추천 콘텐츠 등록부 → 추천 시스템 → 단말 사용자 인터페이스 → 즉시 재생 제어부
3. **광고·권한·재생 사슬:** 광고정보 획득 → 단말 광고 재생 → 단말·플랫폼 광고 확인 → 업체 권한 획득 → 단말 콘텐츠 재생

이 결합형은 권리범위는 좁아지지만, 검증되지 않은 외부 VOD 콘텐츠가 추천·광고·권한 획득·재생 경로로 진입하지 못하는 폐쇄된 기술적 제어구조를 명확히 한다.

### 13.8 종속 청구항 후보

- 광고 시청 확인정보를 세션·콘텐츠·광고 요구시점에 결합하는 구성
- 광고 처리부가 제공한 광고 목록 식별자 또는 버전과 광고 재생결과를 대조하는 구성
- 광고 시청 확인정보에 일회성 값 또는 무결성 증명을 포함하는 구성
- 하나의 광고 시청 확인정보를 하나의 콘텐츠 재생권한 획득 요청에만 사용하는 구성
- 콘텐츠 최초 재생 전과 콘텐츠 재생 중 하나 이상의 광고 요구시점마다 새로운 권한을 획득하는 구성
- 즉시 재생 제어부가 광고 확인 전에는 권한 획득부를 호출하지 않는 구성
- 권한 획득부가 광고 확인정보와 콘텐츠·세션이 불일치하면 VOD 업체에 요청을 전송하지 않는 구성
- 업체별로 서로 다른 권한 서비스 주소·인증방식·요청규격을 저장하는 구성
- 콘텐츠 등록정보가 업체 콘텐츠 식별자와 업체 권한 서비스 식별자를 함께 포함하는 구성
- 등록단계에서 검증용 권한을 획득하고 실제 콘텐츠 재생을 시험하는 구성
- 검증 성공 콘텐츠만 추천 시스템에 등록하고 검증 만료 시 추천 등록을 해제하는 구성
- VOD 업체가 발급한 권한의 발급주체·콘텐츠·세션·범위·만료를 검증하는 구성
- 획득 권한을 특정 재생구간 또는 짧은 유효시간으로 제한하는 구성
- 광고 확인·권한 요청·업체 응답·실제 재생결과를 동일 세션으로 연결하는 구성
- 연결된 기록을 VOD 업체와의 광고수익 배분 근거로 사용하는 구성

## 14. 기술적 차별성과 기술적 효과

### 14.1 기존 기능과 구별하여 주장할 부분

다음 기능 자체는 기존 기술을 사용할 수 있으므로 독립적인 차별점으로 과도하게 주장하지 않는다.

- 콘텐츠 추천 알고리즘
- 광고 선정 및 광고 표시형식
- 일반적인 스트리밍과 멀티미디어 재생
- DRM과 콘텐츠 보호
- VOD 업체의 일반적인 권한 발급
- 광고수익 계산

본 발명에서 강조할 기술적 차별점은 이들 기존 기능 사이의 조건, 데이터 연결 및 제어 순서이다.

### 14.2 차별성 항목

| 차별성 항목 | 기술적 수단 | 기술적 효과 |
|---|---|---|
| 단말·플랫폼 협력형 광고 확인 | 단말의 광고 시청 확인부가 멀티미디어 재생결과를 이용하여 플랫폼 광고 처리부와 완료를 확인 | 단말의 임의 완료상태만으로 콘텐츠 권한 획득이 진행되는 것을 제한 |
| 광고 완료와 업체 권한의 분리 | 광고 확인정보를 재생권한 자체가 아닌 권한 획득 요청의 선행조건으로 사용 | 플랫폼이 VOD 업체 권한 체계를 침해하지 않으면서 광고 기반 이용자격 구현 |
| VOD 업체 발급 권한의 중개 획득 | 저장소의 업체·콘텐츠·권한 서비스 연결정보로 해당 업체에 권한 요청 | 복수 업체의 서로 다른 권한 서비스를 단일 즉시 재생 경험으로 연결 |

| 검증결과 기반 추천 등록 | 활성·검증된 콘텐츠만 추천 시스템에 등록 | 추천 노출과 실제 즉시 재생 가능성의 불일치 감소 |
| 콘텐츠별 권한 서비스 매핑 | 플랫폼 콘텐츠 식별자, 업체 콘텐츠 식별자, 재생 자산 및 권한 서비스의 사전 연결 | 콘텐츠 선택 후 권한 요청대상을 신속하고 결정적으로 식별 |
| 광고 요구시점별 권한 갱신 | 추가 광고 완료마다 다음 재생구간의 한시적 권한을 획득 | 광고 완료 여부에 따라 콘텐츠의 계속 재생을 구간 단위로 제어 |
| 기존 재생기·DRM 재사용 | 즉시 재생 제어부가 단말의 기존 멀티미디어 재생기에 권한과 재생명령 제공 | 새로운 스트리밍·보호기술을 요구하지 않고 기존 VOD 인프라 활용 |

### 14.3 특히 강하게 주장할 기술적 조합

가장 강한 조합은 다음과 같다.

> `검증된 업체·콘텐츠·권한 서비스의 사전 매핑`과 `단말 광고 시청 확인부 및 플랫폼 광고 처리부의 완료 확인`을 결합하고, 그 확인 후에만 `플랫폼 권한 획득부가 VOD 업체 발급 권한을 획득`하여 `단말의 기존 재생기`에 제공하는 구조.

이 조합은 단순한 광고부가 VOD를 재생하는 구조와 다르다. 또한 플랫폼이 자체 이용권을 발급하는 구조와도 다르다. 광고 완료 확인은 플랫폼의 권한 획득 절차를 개시하는 조건이고, 실제 콘텐츠 사용허가는 등록된 VOD 업체가 발급한다.

### 14.4 선행기술 대비 검토 시 확인할 포인트

실제 신규성·진보성 판단은 선행기술 조사 후 확정해야 한다. 조사와 명세서 작성 시 다음 결합이 하나의 문헌에 모두 개시되어 있는지를 중점적으로 확인한다.

1. 외부 VOD 업체의 콘텐츠와 권한 요청방법을 플랫폼에 사전 등록하는가.
2. 등록단계에서 플랫폼이 검증용 권한을 획득하여 실제 재생을 시험하는가.
3. 검증된 콘텐츠만 별도 추천 시스템에 등록하는가.
4. 사용자 단말에 광고 시청 확인부와 즉시 재생 제어부가 분리되어 있는가.
5. 단말 광고 시청 확인부가 플랫폼 광고 처리부와 광고 완료를 확인하는가.
6. 단말 제어부가 광고 완료 확인 후에만 플랫폼 권한 획득부를 호출하는가.
7. 플랫폼이 자체 권한을 발급하지 않고 콘텐츠 업체가 발급한 한시적 권한을 획득하는가.
8. 획득 권한을 단말의 기존 멀티미디어 재생기에 제공하여 앱·업체 로그인 없이 재생하는가.
9. 추가 광고마다 다음 콘텐츠 구간의 권한 획득을 반복하는가.

개별 요소가 알려져 있더라도 위 요소들의 특정 소속, 연결 방향 및 조건부 실행관계가 결합되어 있지 않다면 결합구조와 기술적 효과를 중심으로 진보성을 주장할 수 있다.

## 15. 요약

즉시 재생의 핵심은 새로운 광고기술, 추천기술, 스트리밍 기술 또는 DRM이 아니다.

핵심은 다음 구조이다.

~~~text
VOD 업체·콘텐츠·권한 요청방법 등록
→ 검증용 권한 획득과 실제 콘텐츠 재생 검증
→ 검증된 콘텐츠만 저장·추천 등록
→ 사용자가 단말 사용자 인터페이스에서 콘텐츠 선택
→ 단말 즉시 재생 제어부가 플랫폼 광고 처리부에서 광고 목록 획득
→ 단말 멀티미디어 재생기로 광고 재생
→ 단말 광고 시청 확인부가 플랫폼 광고 처리부와 광고 완료 확인
→ 광고 확인 후에만 플랫폼 콘텐츠 재생권한 획득부 호출
→ 저장소의 업체·콘텐츠 연결정보로 VOD 업체에 권한 요청
→ VOD 업체가 발급한 한시적 콘텐츠 재생권한 획득
→ 단말 즉시 재생 제어부가 멀티미디어 재생기로 VOD 콘텐츠 재생
~~~

> 필수 광고의 시청 완료가 확인되지 않으면 콘텐츠 재생권한 획득부는 VOD 업체 시스템에 사용자 재생용 권한을 요청하지 않는다. VOD 업체가 콘텐츠 재생권한을 발급하지 않으면 즉시 재생 제어부는 멀티미디어 재생기로 콘텐츠를 시작하거나 계속 재생시키지 않는다.
